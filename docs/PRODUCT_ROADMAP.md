# 백테스트K 제품 로드맵

## 제품 원칙

한국판 Portfolio Visualizer의 핵심은 기능 수를 복제하는 것이 아니라, 한국 투자자의 실제 의사결정 단위인 **원화·적립식·계좌·세금·환율·국내 상장 ETF**를 일관된 데이터와 산식으로 연결하는 것입니다.

| 영역 | 구현 상태 |
|---|---|
| 포트폴리오 백테스트 | 구현 |
| 적립식 현금흐름 | 구현 |
| 주기별 리밸런싱·거래비용 | 구현 |
| TWRR·MWRR·MDD·위험지표 | 구현 |
| 벤치마크·연도별 수익률 | 구현 |
| 위험기여도·상관관계 | 구현 |
| 몬테카를로 | 구현 |
| 자산 비교 | 구현 |
| CSV 데이터 가져오기 | 구현 |

## Phase 1 — 신뢰 가능한 한국 ETF 백테스트

목표: 사용자가 종목코드만 입력하면 실제 데이터를 이용한 재현 가능한 결과를 얻도록 합니다.

- ETF·주식·지수 마스터 데이터
- 거래일 캘린더와 월말 스냅샷
- 수정주가·총수익지수·분배금 분리 저장
- 원/달러·원/엔·원/유로 환율 시계열
- 환노출·환헤지 상품 메타데이터
- 상장일·상장폐지·합병·종목코드 변경 이력
- 데이터 기준일·원천·버전 표시
- 백테스트 계산 API와 회귀 테스트

권장 구조:

```text
공식/계약 데이터 → 수집 작업 → 원천 저장소 → 조정·검증 ETL
→ 월 수익률/총수익 데이터마트 → 계산 API → 웹 UI
```

## Phase 2 — 한국 투자자 특화 시뮬레이션

- 일반계좌·ISA·연금저축·IRP 프리셋
- 국내상장 해외ETF와 해외직접투자 비교
- 환전비용·배당원천징수·매매차익 과세 입력
- 정액·정률 인출과 은퇴 성공확률
- 물가연동 지출, 국민연금·퇴직연금 현금흐름
- 리밸런싱 밴드, 현금흐름 우선 리밸런싱
- 분배금 재투자/현금수령 비교

세금은 법령과 금융상품 구조 변화가 잦으므로 화면 로직에 직접 박지 않고, 기준일과 근거가 있는 버전형 규칙 엔진으로 분리해야 합니다.

## Phase 3 — 분석 고도화

- 평균분산 최적화와 효율적 투자선
- 최소분산·위험균형·최대분산효과 포트폴리오
- Black–Litterman
- 목표수익·MDD·CVaR 제약 최적화
- 이동창·워크포워드 테스트
- 금리·환율·인플레이션·위기 국면 스트레스 테스트
- 국내/글로벌 팩터 회귀
- 수익률 기여도, 환율 기여도, 비용 기여도 분해
- 전략 간 중복 보유·실질 익스포저 분석

## Phase 4 — 제품화와 수익화

| 요금제 | 대상 | 핵심 기능 |
|---|---|---|
| Free | 개인 탐색 사용자 | 제한된 기간·자산 수, 기본 백테스트 |
| Plus | 적극적 개인투자자 | 저장, 세금·계좌, 고급 시뮬레이션, PDF/CSV |
| Pro | ETF 컨설턴트·콘텐츠 사업자 | 화이트라벨 보고서, 비교 템플릿, API 할당량 |
| Team/API | 금융사·핀테크 | 조직 워크스페이스, 감사로그, 임베드·API |

광고보다 유료 저장·보고서·전문가 워크플로가 제품과 이해관계가 잘 맞습니다. 무료 계산기는 검색 유입과 신뢰 형성 역할을 담당합니다.

## 우선순위 제안

1. 데이터 정확성과 결과 재현성
2. 한국형 계좌·세금·환율
3. 포트폴리오 저장과 공유 링크
4. 은퇴·목표 기반 시뮬레이션
5. 최적화와 팩터 분석
6. B2B 보고서와 API

초기에는 기능 확장보다 “특정 종목·기간·설정으로 동일 결과가 반복되는가”를 핵심 품질지표로 두는 것이 적절합니다.
